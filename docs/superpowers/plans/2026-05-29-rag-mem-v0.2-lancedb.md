# rag-mem v0.2 — LanceDB vector store + tag filtering (slice 1)

Date: 2026-05-29
Status: in progress

## Goal
Replace the in-memory NumPy cosine vector store with an embedded, on-disk **LanceDB**
table behind the unchanged `KnowledgeBase` facade, and add **tag filtering** to
`semantic_search`. First slice of v0.2. One commit; suite green at the boundary.

## Decisions (confirmed with user 2026-05-29)
- **Scope = swap + tags.** Both the NumPy→LanceDB swap and the `tags` field/filter land
  in this slice.
- **Replace NumPy fully.** Delete `ragmem/index/semantic.py` (NumPy cosine) and the
  `embeddings.npy` matrix cache. LanceDB self-persists the vectors on disk. Keep
  `content_hash` + a freshness manifest so the table rebuilds when source `.md` changes.
- LanceDB is **embedded/local** (like `bm25s`) → tests use a **real** LanceDB table on
  `tmp_path`; only the OpenAI embedder is mocked (the network-free-pytest rule holds).

## Decisions (my call — flagged to user, easy to flip)
- **Tag semantics = OR.** `tags=["a","b"]` matches chunks having *any* of the tags
  (`array_has(tags,'a') OR array_has(tags,'b')`), applied as a **prefilter** so the
  k-NN limit is taken over matching rows. Single tag is unambiguous.
- **Flat (exact) search**, no ANN index this slice — correct + deterministic for current
  corpus sizes; `create_index` (IVF/HNSW) is documented future work for millions-scale.
- **Tag *population* deferred.** This slice builds the `tags` field + filter plumbing;
  chunks carry `tags=()` until a follow-up derives them (e.g. YAML frontmatter).
- Version → **0.2.0**.

## Storage layout (per knowledge base, under `<kb>/.ragmem/`, gitignored)
- `vectors.lance/` — LanceDB table: id, vector(fixed_size_list<f32,dim>), doc_path,
  heading_path(list<str>), text, start_line, end_line, tags(list<str>).
- `manifest.json`  — freshness marker: content_hash, embedding_model, chunk_ids.
- `graph/`         — Kuzu graph (unchanged).

## Module shape
- NEW `ragmem/index/lance.py` — `LanceSemanticIndex(table, embedder)` with
  `build(chunks, embedder, db)`, `open(db, embedder)`, `search(query, k=5, tags=None)`.
  Score = `1 - _distance` (cosine). Empty corpus → no table; search returns [] without
  embedding the query.
- `ragmem/types.py` — `Chunk.tags: tuple[str, ...] = ()`.
- `ragmem/persistence.py` — keep `content_hash`; add `save_manifest` / `manifest_fresh`.
- `ragmem/knowledge_base.py` — DI a `vector_db` (default `lancedb.connect(persist_dir)`),
  build/open the Lance index with freshness check; `semantic_search(query, k, tags=None)`.
- `ragmem/cli.py` — repeatable `--tag` on `search`, passed through in semantic mode.
- `pyproject.toml` — add `lancedb`, `pyarrow`; bump to 0.2.0.

## Tests
- `tests/test_lance.py` (real LanceDB + FakeEmbedder): nearest-first ordering; cosine
  score≈1 for identical vectors; k clamped; empty corpus returns [] without embedding;
  breadcrumb text indexed; tag filter restricts; multi-tag OR; tags=None unfiltered;
  persist + reopen; source=="semantic".
- `tests/test_persistence.py` rewrite: content_hash; manifest_fresh true/false on
  missing / text-changed / model-changed / chunkset-changed.
- `tests/test_knowledge_base.py`: StubIndex.search gains tags=None; cache-reuse /
  invalidation tests now exercise real LanceDB + manifest.
- `tests/test_cli.py`: FakeKB.semantic_search tags=None; `--tag` pass-through.
- Delete `tests/test_semantic.py`.
